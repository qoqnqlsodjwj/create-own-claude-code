## 第一步：什么是 AI Agent？

### Agent 的本质公式

AI Agent 的本质可以用一个简单的公式概括：

```
Agent = LLM + 工具 (Tools) + 循环 (Loop)
```

- **LLM**：大脑，负责理解、推理和决策
- **工具**：手和脚，负责与外部世界交互
- **循环**：自主性，LLM 可以多轮调用工具直到任务完成

### Agent 和普通 LLM 调用的区别

```
普通 LLM 调用:
  用户: "src/index.ts 有什么问题？"
  LLM: "我无法读取你的文件，请把内容粘贴过来。"

Agent 调用:
  用户: "src/index.ts 有什么问题？"
  LLM: "让我读取这个文件。" → 调用 read_file("src/index.ts")
  工具返回文件内容
  LLM: "我发现以下问题：第 10 行有类型错误..."
```

## 第二步：Claude Code 的 AgentTool 设计

### AgentTool 是什么？

Claude Code 中有一个非常精妙的设计——**AgentTool**。它把 Agent 包装成一个"工具"，让主 Agent 可以像调用普通工具一样，派生出子 Agent 来执行子任务。

### 上下文隔离——AgentTool 的关键特性

子 Agent 看不到主 Agent 的完整对话历史：

```
主 Agent 的对话历史 (可能很长):
  [system] [user msg 1] [assistant 1] [tool 1] ... [user msg 50]
  ↓
  子 Agent 只看到:
  [system_prompt + task_description]
```

为什么要隔离？

1. **节省 Token**：不需要把整个对话历史传给子 Agent
2. **提高质量**：子 Agent 专注于一个明确的任务
3. **并行安全**：多个子 Agent 之间互不影响

## 第三步：进程内 vs 子进程 Agent

### 进程内 Agent（In-Process）

```
┌─────────────────────────────────────┐
│           Node.js 进程               │
│                                      │
│   ┌──────────────────────────────┐  │
│   │         Lead Agent            │  │
│   │         (同一个进程)           │  │
│   └──────────┬───────────────────┘  │
│              │                       │
│   ┌──────────┴───────────────────┐  │
│   │   Worker 1    Worker 2       │  │
│   │  (同进程)     (同进程)        │  │
│   └──────────────────────────────┘  │
└─────────────────────────────────────┘
```

优点：通信快、实现简单、共享状态方便
缺点：不隔离、不能利用多核

### 子进程 Agent（Out-of-Process）

```
┌──────────────┐     ┌──────────────┐
│  Lead Agent   │     │  Worker 1    │
│  (进程 1)     │←──→│  (进程 2)    │
└──────────────┘     └──────────────┘
```

优点：完全隔离、可利用多核
缺点：通信需要序列化、实现更复杂

## 第四步：多智能体协作

### Swarm 架构

```
┌─────────────────────────────────────────────────┐
│                 Swarm 架构                        │
│                                                   │
│              ┌──────────┐                        │
│              │  Lead     │                        │
│              │  Agent    │                        │
│              └────┬─────┘                        │
│                   │                               │
│         ┌────────┼────────┐                      │
│         v        v        v                      │
│    ┌────────┐ ┌────────┐ ┌────────┐             │
│    │Worker 1│ │Worker 2│ │Worker 3│             │
│    │搜索代码 │ │写代码  │ │跑测试  │             │
│    └────────┘ └────────┘ └────────┘             │
└─────────────────────────────────────────────────┘
```

### 任务队列

- Lead 创建任务并添加到队列
- Worker 从队列认领待处理任务
- Worker 完成后更新队列中的任务状态

## 关键概念总结

| 概念 | 说明 |
|------|------|
| Agent | LLM + 工具 + 循环 = 自主执行任务的实体 |
| AgentTool | 将 Agent 包装为工具，让主 Agent 可以派生子 Agent |
| 上下文隔离 | 子 Agent 看不到主 Agent 的完整对话历史 |
| 专业化 Agent | 为不同任务定制不同的工具集、提示词和行为 |
| Swarm | 多 Agent 协作模式：Lead 分发、Worker 执行 |
| TaskQueue | 任务队列管理多 Agent 的任务分发 |
