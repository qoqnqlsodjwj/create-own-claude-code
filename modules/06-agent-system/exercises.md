# 模块 06: Agent 与多智能体系统 - 练习题

## 练习 1: 实现自定义 Agent（基础）

### 目标
创建一个专门用于代码重构的 Refactor Agent。

### 要求

1. 创建一个 `RefactorAgent` 类，继承 `InProcessAgent`
2. Agent 应该具备以下能力：
   - 识别代码中的代码异味（Code Smell）
   - 提出重构建议
   - 执行安全重构
3. 使用只读工具来分析代码，使用读写工具来执行重构
4. 添加重构成败的记录和报告

### 验收标准
- [ ] Agent 能识别至少 3 种代码异味
- [ ] Agent 能提供重构建议并解释原因
- [ ] Agent 能执行至少 2 种类型的重构

## 练习 2: 实现子进程 Agent 通信（进阶）

### 目标
实现一个可靠的子进程 Agent IPC 机制。

### 要求

1. 扩展 `SpawnAgent` 类
2. 实现以下功能：
   - **进度报告**：子进程定期向主进程报告进度
   - **心跳检测**：检测子进程是否存活
   - **优雅关闭**：支持 SIGTERM 和 SIGKILL
   - **错误重试**：可配置的重试机制

### 验收标准
- [ ] 支持进度报告
- [ ] 支持心跳检测
- [ ] 优雅关闭机制工作正常

---

## 练习 3: 实现任务依赖图（高级）

### 目标
实现一个任务依赖图，支持拓扑排序和关键路径分析。

### 要求

1. 扩展 `TaskQueue` 类
2. 实现以下功能：
   - **添加带依赖的任务**：支持 DAG
   - **检测循环依赖**：防止死锁
   - **拓扑排序**：按正确顺序执行任务
   - **关键路径分析**：识别最长的执行路径

### 验收标准
- [ ] 支持添加带依赖的任务
- [ ] 能检测并报告循环依赖
- [ ] 按拓扑顺序返回可执行的任务

## 练习 4: 完整项目 - 智能代码审查团队（挑战）

### 目标
构建一个由多个专业 Agent 组成的代码审查团队。

### 团队架构
```
代码审查团队
├── Orchestrator (协调者)
│   ├── CodeAgent (代码分析)
│   ├── SecurityAgent (安全审查)
│   └── DocAgent (文档审查)
```

### 功能要求

1. **协调者**：
   - 接收代码审查请求
   - 分析代码类型
   - 分配给合适的审查 Agent
   - 汇总审查结果

2. **代码分析 Agent**：代码风格、最佳实践、复杂度分析

3. **安全审查 Agent**：漏洞检测、敏感信息检测

4. **文档审查 Agent**：API 文档、注释覆盖率

### 验收标准

- [ ] 所有 Agent 都能正常工作
- [ ] Orchestrator 能正确分配任务
- [ ] Agent 之间能有效通信
- [ ] 生成结构化的审查报告
