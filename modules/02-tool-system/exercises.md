## 练习 1: 实现 GrepTool（搜索文件内容）

### 目标
实现一个能在文件中搜索指定文本的工具。

### 要求
1. 实现一个 `GrepTool`，满足 `Tool` 接口
2. 支持参数:
   - `pattern` (必需): 搜索的正则表达式模式
   - `path` (可选): 搜索的目录路径，默认当前目录
   - `filePattern` (可选): 文件名过滤 (如 `*.ts`)
3. 递归搜索目录下所有匹配的文件
4. 返回匹配的行，带文件名和行号

### 提示
- 使用 `fs.readdir` 递归遍历目录
- 使用 `RegExp` 来匹配文本
- 输出格式建议: `文件路径:行号: 匹配的行内容`

### 验证
创建几个测试文件，搜索特定内容，确认结果正确。

## 练习 2: 实现 GitTool

### 目标
实现一个 Git 操作工具。

### 要求
1. 实现 `GitTool`，满足 `Tool` 接口
2. 支持参数:
   - `command` (必需): git 命令（如 "status"、"log"、"diff"）
   - `repoPath` (可选): 仓库路径
3. 封装常用的 git 命令
4. 返回命令输出

### 提示
- 底层调用 `git` 命令
- 可以用 spawn 执行

## 练习 3: 添加工具缓存

### 目标
为工具执行添加缓存机制。

### 要求
1. 创建一个缓存中间件
2. 根据工具名称和输入参数生成缓存键
3. 缓存有效期可配置
4. 缓存命中时直接返回结果

### 提示
- 使用 Map 存储缓存
- 可以用工具名 + JSON.stringify(input) 作为键

## 练习 4: 实现日志记录

### 目标
为工具执行添加详细的日志记录。

### 要求
1. 记录每次工具调用的:
   - 工具名称
   - 输入参数
   - 执行时间
   - 执行结果
   - 错误信息（如果有）
2. 日志级别支持: debug, info, warn, error
3. 可配置的日志输出（控制台、文件）

### 提示
- 实现一个日志中间件
- 在执行前后记录关键时间点

## 练习 5: 添加超时处理

### 目标
为所有工具添加统一的超时处理。

### 要求
1. 创建一个超时中间件
2. 可为每个工具单独设置超时时间
3. 超时时返回统一的错误格式
4. 支持清理超时任务

### 提示
- 使用 Promise.race 实现超时
- 超时后需要清理可能的后台任务

## 挑战练习: 实现工具组合

### 目标
实现一个"宏"工具，可以组合多个工具调用。

### 要求
1. 创建 `MacroTool`
2. 输入是一个工具调用序列
3. 按顺序执行每个工具
4. 将所有结果汇总返回

### 示例输入
```json
{
  "macro": [
    { "tool": "read", "input": { "path": "config.json" } },
    { "tool": "bash", "input": { "command": "echo $VALUE" } }
  ]
}
```

### 这个练习将帮助你:
- 理解工具系统的可组合性
- 体验真实的 AI 编程助手的工作流程
